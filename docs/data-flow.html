Data Flow

How a log analysis request travels through the system end-to-end

1
User selects analysis source in Dashboard
Synthetic · System logs · Upload · GitHub repo URL
POST /analyze · POST /analyze/upload · POST /analyze/github
↓
2
FastAPI creates SecurityState + starts SSE stream
Returns session_id immediately. Dashboard opens GET /stream/{session_id} (SSE).
SecurityState = { logs[], anomalies[], cve_matches[], vulnerabilities[], action_plan[], compliance_gaps[] }
↓
3
📋 Log Monitor Agent runs
Parses raw logs → detects anomalies → classifies severity
Emits SSE: { agent: "log_monitor", status: "running" | "done", findings: [...] }
↓ state.anomalies populated
4
🔍 Threat Intel Agent runs
Reads anomalies → queries NVD API → checks IP reputation → maps MITRE ATT&CK
Emits SSE: { agent: "threat_intel", cve_matches: [...], threat_score: 0-100 }
↓ state.cve_matches + threat_score populated
5
🛡️ Vuln Scanner Agent runs
OWASP mapping · HTTP headers (log runs) · GitHub static scan (Terraform/IaC patterns)
Emits SSE: { agent: "vuln_scanner", status: "done" } → code_findings[], risk_level
↓ state.vulnerabilities populated
6
🚨 Incident Response Agent runs
OpenRouter gpt-4o via CachingLLMClient (LRU cache) · fallback action plan if LLM unavailable
Emits SSE: { agent: "incident_response", status: "done" } → action_plan[], runbook_md
↓ state.action_plan + runbook populated
7
📜 Policy Checker Agent runs
Maps all findings to NIST/SOC2/ISO gaps → generates compliance score
Emits SSE: { agent: "policy_checker", compliance_gaps: [...], score: 72 }
↓ analysis complete
8
Dashboard renders final report
Metrics · Agent pipeline boxes · Threats & Remediation panel · Incident report · Evals tab
GET /report/{session_id} · GET /evals/{session_id}
