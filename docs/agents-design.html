Agent Design

Each agent's role, tools, inputs, and outputs in the shared SecurityState

📋 Log Monitor Agent
TOOLS
parse_logs(source)
detect_anomalies(entries)
classify_severity(event)
DETECTS
SSH brute force, port scans
Path traversal, sudo failures
Enriched with title + recommendation
OUTPUT → STATE
anomalies[] (title, recommendation), severity_map{}
🔍 Threat Intelligence Agent
TOOLS
query_nvd_cve(keyword)
lookup_ip_reputation(ip)
match_attack_patterns(anomaly)
DATA SOURCES
NVD API (real CVE data)
MITRE ATT&CK patterns
AbuseIPDB for IP lookups
OUTPUT → STATE
cve_matches[], threat_score
🛡️ Vulnerability Scanner Agent
TOOLS
scan_for_owasp(anomalies)
check_api_headers() — log runs only
scan_github_repo_safe(url)
CHECKS
OWASP Top 10 · Terraform/IaC patterns
Missing security headers
Hardcoded secrets, open CIDRs, public S3
OUTPUT → STATE
vulnerabilities[], code_findings[]
risk_level, files_scanned, scan_error
🚨 Incident Response Agent
TOOLS
call_openai via CachingLLMClient
_fallback_action_plan()
READS FROM STATE
anomalies, cve_matches, code_findings
vulnerabilities, threat_score
OUTPUT → STATE
action_plan[], runbook_md
📜 Policy Checker Agent
TOOLS
map_to_nist · map_to_soc2
map_code_findings_to_compliance()
STANDARDS
NIST CSF 2.0 (anomalies + code)
SOC 2 Type II controls
OUTPUT → STATE
compliance_gaps[], score%
